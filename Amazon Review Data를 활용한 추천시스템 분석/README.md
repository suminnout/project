## 📊 EDA (Exploratory Data Analysis)

### 1. 평점 분포 분석
- 전체적으로 **4~5점 고평점에 편중된 분포**
- 리뷰 기반 서비스 특성상 긍정적 평가 비율이 높음

### 2. 사용자 및 상품 분포
- 소수의 상품에 리뷰가 집중되는 **롱테일(Long-tail) 구조**
- 사용자별 리뷰 수 편차가 큼

### 3. 사용자 군집 분석
- K-means 기반 사용자 군집화 (K=4)
- 사용자 평점 성향에 따라 다음과 같은 그룹 도출
  - 고평점 위주 사용자
  - 저평점에 민감한 사용자
  - 평균적 소비 성향 사용자

## 🤖 추천 시스템 구현

### Model 1: 아이템 기반 협업 필터링 (Pearson Similarity, K=5)

- **설명**  
  아이템 간 공통 사용자 평점을 기반으로 **피어슨 상관계수**를 계산하여 추천을 수행하는 모델입니다.
- **특징**
  - 평점의 상대적 경향을 보정 가능
  - 아이템 수가 상대적으로 적을 때 효율적

---

### Model 2: 아이템 기반 협업 필터링 (Pearson Similarity, K=10)

- **Model 1 확장**
  - 이웃 아이템 수를 5 → 10으로 증가
- **분석 목적**
  - K 증가가 Recall과 Precision에 미치는 영향 분석

---

### Model 3: 아이템 기반 협업 필터링 (Cosine Similarity, K=5)

- **설명**  
  아이템 간 유사도를 **코사인 유사도**로 계산한 협업 필터링 모델입니다.
- **특징**
  - 평점의 절대 크기를 포함
  - Pearson 방식과의 성능 차이 비교 목적

---

### Model 4: 아이템 기반 협업 필터링 (Cosine Similarity, K=10)

- **Model 3 확장**
  - 이웃 수 증가에 따른 추천 안정성 및 성능 변화 분석
- **분석 포인트**
  - 유사도 방식보다 K 값의 영향 확인

---

### Model 5: 아이템 기반 협업 필터링 + 인기 보정 모델

- **설명**  
  협업 필터링 결과에 **상품 인기 지표(평점 수)** 를 결합한 하이브리드 모델입니다.
- **도입 이유**
  - 협업 필터링의 데이터 희소성 문제 완화
  - 신뢰도 높은 인기 상품 반영
- **특징**
  - RMSE 개선 확인
  - 실제 서비스 환경에 가까운 추천 구조

---

## 📈 모델 성능 평가

| Model | Similarity | K | Precision | Recall | NDCG | RMSE |
|------|------------|---|-----------|--------|------|------|
| Model 1 | Pearson | 5 | 0.1632 | 0.7814 | 0.7797 | 1.3106 |
| Model 2 | Pearson | 10 | 0.0816 | 0.7815 | 0.7798 | 1.3106 |
| Model 3 | Cosine | 5 | 0.1632 | 0.7814 | 0.7797 | 1.3128 |
| Model 4 | Cosine | 10 | 0.0816 | 0.7815 | 0.7798 | 1.3128 |
| Model 5 | Cosine + Popularity | 10 | 0.0816 | 0.7815 | 0.7801 | 1.2505 |

---

## 🔎 결과 해석 및 인사이트

- K 값 증가 시 Recall은 유지되나 Precision 감소
- 유사도 방식(Pearson vs Cosine)보다 **이웃 수(K)가 성능에 더 큰 영향**
- 인기 보정 모델은 RMSE를 개선하며 추천 안정성을 강화
- 추천 시스템은 단일 지표가 아닌 **다양한 평가 지표의 균형 해석이 중요**
