본 프로젝트는 K리그 구단 행정 업무에서 발생하는  
**규정 탐색, 문서 검색, 반복 질의 응답의 비효율 문제를 해결하기 위해  
RAG(Retrieval-Augmented Generation) 기반 AI 행정 어시스턴트를 설계·구현한 프로젝트**입니다.

단순한 챗봇이 아닌,  
**실제 행정 실무를 보조하는 AI 서비스**를 목표로 문제 정의부터 기술 구조, 확장 가능성까지 종합적으로 설계했습니다.

## 🎯 문제 인식 (Why)

### K리그 구단 행정의 현실적 문제

1. **관중 수 증가 → 행정 규정 복잡도 증가**
2. 실무 담당자의 **업무 과부하 및 인력 한계**
3. 규정 확인 방식이 여전히 **PDF 검색(Ctrl+F) 중심**
4. 국내 스포츠 행정 분야에 **AI 기반 실무 보조 시스템 부재**

> 즉, 정보는 존재하지만  
> **“필요한 순간에, 맥락에 맞게 찾기 어렵다”는 구조적 문제가 존재**

## 🧠 해결 아이디어

### RAG 기반 행정 AI 어시스턴트

- 규정·문서·내부 자료를 **VectorDB로 구조화**
- LLM이 임의로 답변하지 않고 **근거 문서를 검색 후 응답**
- 단순 Q&A가 아닌 **행정 맥락 기반 질의 응답 지원**

---

## 🛠 서비스 구현 기술

### 핵심 기술 구성

#### 1️⃣ Vector Database
- 구단 행정 규정, 문서, 매뉴얼 임베딩 저장
- 의미 기반 검색으로 키워드 의존성 제거
- 최신 규정 업데이트 용이

#### 2️⃣ RAG (Retrieval-Augmented Generation)
- LLM 단독 응답의 **Hallucination 문제 해결**
- 검색 → 재정렬 → 응답 생성 구조
- 규정 기반 신뢰도 높은 답변 제공

#### 3️⃣ LangGraph
- 질의 유형에 따른 **멀티 워크플로우 분기**
- 단순 Q&A / 업무 보조 / 규정 해석 등 역할 분리
- 확장 가능한 에이전트 구조 설계

#### 4️⃣ MCP (Multi-Component Pipeline)
- 문서 검색, 재정렬, 응답 생성을 모듈화
- 유지보수 및 기능 확장 고려한 파이프라인 설계

## 🔄 서비스 워크플로우

1. 사용자 질의 입력  
2. 질의 유형 분류 (규정 / 업무 / 일반)  
3. VectorDB 기반 문서 검색  
4. 문서 재정렬 (Re-ranking)  
5. LLM 응답 생성  
6. 근거 문서 기반 답변 제공
